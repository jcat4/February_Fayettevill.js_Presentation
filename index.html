What if this data could be changed in multiple places?
Suddenly, both solutions fall short.
Components (plural) that mutate:
  1. When data changes, emit a global event with new data
  2. When event received, replace old data with new data

Other components reliant on data:
  1. When event received, replace old data with new data
When should I use state management?

  1. Share data between components in an easier way
  2. If your data needs to "jump" up or down the component tree by more than one hop
  3. Persist data outside the lifetime of a component
  4. Clean and powerful way to maintain state
